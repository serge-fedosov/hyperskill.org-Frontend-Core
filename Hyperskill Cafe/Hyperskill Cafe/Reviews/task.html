Description

In this stage, we will design the reviews section. It should contain comments and reviews from clients. So far, you have been using HTML and CSS to render the content on the website. Now, you need to add JavaScript into the mix and render new comments and reviews dynamically. Add three rendered reviews (see Example) and allow new ones under the <form> and <input> tags. For the <form>, you can use the Bootstrap form component.

Once visitors submit the form, a new review should be posted on the website. You don't have to display the reviews permanently; they can be removed when you refresh the page. This means that you need to use a submit event in the JavaScript action listener.

When using the submit event, don't forget to use event.preventDefault() function to prevent the page from refreshing. Otherwise, the tests will not work properly.

To select input tags from documents in your script file with ease, add these IDs to the following tags:

Reviews Row Div:   id = "reviewsRow" 

Review Title Input: id="reviewTitle"

Review Text Input: id="reviewText"

Review Name Input: id="reviewName"

Review Form: id="reviewForm"

Review Button: id="reviewButton"

Put the form component inside the collapse component from Bootstrap so that the form can be toggled on or off.

Just like the image gallery before, nest the comments in a card component with a grid system.

Create a separate script.js and write your code there and then, link the JavaScript file to your HTML file using a <script> tag.

Use the <blockquote> tag for reviews of the customers.

If the users don't enter a name, post the review under Anonymous.

Also, if the users don't enter a title or a comment, the browser must show that those fields are required.

Keep in mind that there is no limitation on the number of reviews that a user can post.

Objectives

In this stage, your website should contain:

  1. A <h2> tag with Reviews From Our Customers;
  2. A form for writing new reviews;
  3. A submit button;
  4. A collapse area with a button to open/close the form;
  5. New reviews wrapped in the card component;
  6. Three initially rendered, static reviews;
  7. New dynamically rendered reviews;
  8. New script.js file;
  9. A <script> tag for your JS file;
  10. Fallback Anonymous value under empty name input;
  11. A required attribute for title and comment inputs;
  12. Correct ID attributes for necessary tags.
